# TABCRYPT backend

Flask + SQLite tab vault for the TabCrypt app. Relevance-scored search, CORS enabled,
optional API key for writes. One file (`app.py`) so it’s easy to edit from mobile and deploy on Railway.

## Files

- `app.py` — the whole service
- `requirements.txt` — Flask + gunicorn
- `Procfile` — gunicorn start command for Railway
- `.gitignore` — keeps the db file and venv out of git

## Deploy on Railway

1. Push these files to a GitHub repo (your `mrturay90` account).
1. Railway → **New Project → Deploy from GitHub repo** → pick the repo.
1. Railway auto-detects Python, installs `requirements.txt`, runs the `Procfile`. No build config needed.
1. (Optional but recommended) Add a variable `API_KEY` = some secret string. With it set, only requests sending header `X-API-Key: <that value>` can add or delete tabs. Reads stay public.
1. Open the generated URL and hit `/api/health` — you should get `{"status":"ok"}`.

### Make the data survive restarts

Railway’s container filesystem is **ephemeral** — the SQLite file resets on redeploy. Two options:

- **Attach a Volume** in Railway, mount it at e.g. `/data`, and set variable `DB_PATH=/data/tabcrypt.db`.
- Or move to Railway **Postgres** later (the schema is trivial to port). SQLite is fine to start.

## Run locally

```
pip install -r requirements.txt
python app.py            # serves on http://localhost:8000
```

## Environment variables (all optional)

|var      |default      |meaning                                               |
|---------|-------------|------------------------------------------------------|
|`PORT`   |8000         |Railway sets this for you                             |
|`DB_PATH`|`tabcrypt.db`|path to the sqlite file (point at a volume to persist)|
|`API_KEY`|unset        |if set, POST/DELETE require header `X-API-Key`        |

## Endpoints

|method|path                                                                 |notes                                     |
|------|---------------------------------------------------------------------|------------------------------------------|
|GET   |`/api/health`                                                        |liveness                                  |
|GET   |`/api/search?q=<text>&filter=all|song|artist|album|tuning|instrument`|list view, no tab content, sorted by match|
|GET   |`/api/tab/<id>`                                                      |full record incl. `content`               |
|POST  |`/api/tab`                                                           |JSON body, creates, returns `{"id": ...}` |
|DELETE|`/api/tab/<id>`                                                      |removes one                               |

### curl

```
# search
curl "https://YOUR-APP.up.railway.app/api/search?q=crawl&filter=all"

# full tab
curl "https://YOUR-APP.up.railway.app/api/tab/demo-1"

# add (include the header only if you set API_KEY)
curl -X POST "https://YOUR-APP.up.railway.app/api/tab" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_SECRET" \
  -d '{"title":"My Riff","artist":"Me","tuning":"Drop A","instrument":"Guitar (7-string)","difficulty":"Advanced","content":"A|--0-0-0--|"}'

# delete
curl -X DELETE "https://YOUR-APP.up.railway.app/api/tab/t-1700000000000" -H "X-API-Key: YOUR_SECRET"
```

## Wiring the frontend to this

The frontend already uses this exact shape, so it’s a small swap in `TabCrypt.jsx`. Drop this helper near the top:

```js
const API = "https://YOUR-APP.up.railway.app"; // your Railway URL
const API_KEY = ""; // set if you enabled API_KEY on the backend

async function apiSearch(q, filter) {
  const r = await fetch(`${API}/api/search?q=${encodeURIComponent(q)}&filter=${filter}`);
  return (await r.json()).results;
}
async function apiGetTab(id) {
  const r = await fetch(`${API}/api/tab/${id}`);
  return await r.json();
}
async function apiAddTab(tab) {
  const r = await fetch(`${API}/api/tab`, {
    method: "POST",
    headers: { "Content-Type": "application/json", ...(API_KEY ? { "X-API-Key": API_KEY } : {}) },
    body: JSON.stringify(tab),
  });
  return await r.json();
}
async function apiDeleteTab(id) {
  await fetch(`${API}/api/tab/${id}`, {
    method: "DELETE",
    headers: API_KEY ? { "X-API-Key": API_KEY } : {},
  });
}
```

Then in the `Library` component:

- Replace the `useState(SEED)` + the two `storage` effects with a `useEffect` that calls `apiSearch(query, filter)` whenever `query` or `filter` changes (debounce ~250 ms), storing the result list in state. The list items only carry summary fields now.
- In the result click handler, call `apiGetTab(id)` to fetch `content` before opening the viewer.
- `addTab` → `await apiAddTab(tab)` then re-run the search; `delTab` → `await apiDeleteTab(id)` then re-run.
- The client-side `score()` function can be deleted — the server sorts.

Everything else (tuner, theme, viewer, add form UI) stays as is.
